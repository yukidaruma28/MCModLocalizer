# MCModLocalizer

Minecraft Mod の `en_us.json` を LLM で日本語化 (`ja_jp.json`) し、リソースパックとして書き出す Flet 製 GUI アプリです。色コード・プレースホルダーを保護したまま翻訳し、既存訳を尊重して不足分のみを補完します。

加えて、**JAR の en_us.json には収まらない Mod 設定ファイル(クエストブック等)** や、**`ja_jp` を認識しない Mod の JAR にロケールを注入する** ための補助 CLI ツールを `tools/` 配下に同梱しています。

> このリポジトリは [norakurage/MCModLocalizer](https://github.com/norakurage/MCModLocalizer) からの fork で、以下を追加した派生版です:
> - **Claude 定額プラン (Pro/Max) での翻訳機能** (Claude Agent SDK 経由)
> - **自動再開ループ** + **レート制限即時中断**
> - **永続ログ**(`mcmodlocalizer.log`)
> - **CLI ツール群**(`tools/translate_vault_quests.py`, `tools/patch_vault_locale.py`)

## 主な機能

### 翻訳バックエンド (3 種類から選択)

| Provider | 認証方法 | 特徴 |
|---|---|---|
| **Gemini** | `GEMINI_API_KEY` (従量課金) | 安価で高速。デフォルト。 |
| **Claude (API)** | `ANTHROPIC_API_KEY` (従量課金) | 高品質。Anthropic Console で発行したキーを使用。 |
| **Claude (定額/Code SDK)** ✨ | Claude Code CLI のログイン認証 | **Pro/Max 定額プラン枠で動作。API キー不要。** |

`Claude (定額/Code SDK)` モードは [claude-agent-sdk](https://pypi.org/project/claude-agent-sdk/) を経由して、ローカルにインストール済みの Claude Code CLI の認証情報を流用します。**追加課金なしで** Claude を翻訳に使えます。

### その他

- **自動翻訳 & リソースパック生成**: Mod JAR から `en_us.json` を抽出し、選んだ Provider で `ja_jp.json` を生成。`<ModPack名>_localize/` 形式のリソースパックを出力。
- **差分翻訳**: 既存の `ja_jp.json` がある場合は未訳エントリーのみ翻訳。
- **コンテキスト保護**: `%s`、`§a`、`{name}` などの装飾コードやプレースホルダーを自動保護→復元。
- **自動再開ループ** ✨: 1 Mod がエラーで失敗しても他の Mod は処理続行。未完了が残ったら最大 4 周まで自動再ループして最後まで翻訳しきる。
- **レート制限即時中断** ✨: Claude 定額プランの 5 時間 / 週次制限を検知すると即座に全体中断。リトライで時間を浪費しない。進捗は保存されているので、制限解除後に再実行すれば続きから処理。
- **レジューム**: 中断した翻訳の進捗は `output/.resume/` に保存され、再実行で続きから処理。
- **永続ログ** ✨: GUI ログとプロバイダの DEBUG 出力を `<出力フォルダ>/mcmodlocalizer.log` にローテーション付きで記録 (1MB × 5 世代)。アプリを閉じた後も原因究明可能。
- **コスト管理**: 「トークン」タブで使用量・概算コスト・履歴を確認 (定額プラン枠の消費目安にも有用)。
- **複数 Mod 一括処理**: 指定フォルダ内の全 Mod を検出して一括処理。
- **安全なキー管理**: API キーは OS の keyring (Windows 資格情報マネージャー等) に保存。

## 必要条件

- Python **3.10 以上** (3.14 で動作確認済み)
- 使う Provider に応じた以下のいずれか:
  - Gemini API キー ([Google AI Studio](https://aistudio.google.com/app/apikey))
  - Anthropic API キー ([Console](https://console.anthropic.com/settings/keys))
  - **Claude Pro/Max 定額プラン** + [Claude Code CLI](https://docs.claude.com/claude-code) のインストール+ログイン
- インターネット接続

## セットアップ

### 1. リポジトリ取得 + 仮想環境

```powershell
git clone https://github.com/yukidaruma28/MCModLocalizer.git
cd MCModLocalizer
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 2. 依存パッケージのインストール

```powershell
pip install -r requirements.txt
```

> **注**: `pyinstaller` は Python 3.14 未対応のため `requirements.txt` からは除外しています。実行ファイル (.exe) をビルドする場合のみ Python 3.13 以下で `pip install pyinstaller` を別途実行してください。

### 3. (定額プランを使う場合のみ) Claude Code CLI のセットアップ

```powershell
# npm 経由でインストール (公式ドキュメント参照)
npm install -g @anthropic-ai/claude-code

# ログイン (ブラウザが開きます)
claude login

# 動作確認
claude --version
```

ログイン状態は OS グローバルに保持されるので、一度済ませれば以降の起動では不要です。

```powershell
python -m app.main
```

## 使い方

### 1. 設定タブで Provider を選択

| 使いたいバックエンド | 操作 |
|---|---|
| Gemini | Provider = `Gemini` → API キーダイアログから Gemini キーを保存 |
| Claude (従量) | Provider = `Claude (API)` → Anthropic API キーを保存 |
| Claude (定額) | Provider = `Claude (定額/Code SDK)` → **API キー入力は不要** (空欄でキャンセル可) |

モデルは選んだ Provider に応じて自動的に切り替わります:

- Gemini: `gemini-2.5-flash`, `gemini-2.5-flash-lite`
- Claude (API/SDK 共通): `claude-haiku-4-5`, `claude-sonnet-4-6`, `claude-opus-4-7`

### 2. 抽出タブで翻訳実行

1. **Mods フォルダ** を選択 (`.minecraft/mods` など)
2. **リソースパックフォルダ** を選択 (Mods フォルダから自動推測されます)
3. **「抽出 / リソースパック生成」** をクリック

JAR スキャン → 翻訳 → リソースパック書き出しが順次進行します。途中で **「停止」** を押すと現在のバッチ完了後に止まり、進捗は `output/.resume/` に保存されます。

### 3. トークンタブで使用量確認

- 直近の翻訳でのトークン使用量・概算コスト
- 累計トークンとコスト
- 過去の API コール履歴

定額プラン使用時は実際の追加課金は発生しませんが、Claude Code 側の利用量上限の目安として活用できます。

## 永続ログファイル

GUI ログパネルはアプリを閉じると消えますが、同じ内容は `<出力フォルダ>/mcmodlocalizer.log` にも記録されます (出力フォルダ未指定時は `~/.mcmodlocalizer/mcmodlocalizer.log`)。

- **形式**: `[YYYY-MM-DD HH:MM:SS] [LEVEL] message`
- **ローテーション**: 1MB × 5 世代 (`.log`, `.log.1` … `.log.5`)
- **記録内容**:
  - GUI に表示される全ログ ([INFO] / [WARN] / [ERROR] / [STOP])
  - プロバイダの DEBUG 出力 (`--- [DEBUG] SEND User ---` 等)
  - 例外発生時の traceback
  - 起動時のコンテキスト (provider / model)

翻訳が途中で止まった原因を後から追える。bug 報告時に該当ファイルを添付するのが楽。

## 中断・再開・自動再ループの挙動

長尺の翻訳ではレート制限や個別 Mod のエラーが避けられないため、以下の段階的な対応を行います:

### Mod 単位の通常エラー
1 Mod が(レート制限以外の)例外で落ちた場合は、`[INFO] <modid> はスキップして次の Mod に進みます。` とログに残し、**他の Mod は処理続行**。1 Mod の失敗で全体が止まらない。

### レート制限ヒット時 — 即時中断
Claude 定額プランの 5 時間 / 週次制限を検知すると、リトライせず **即座に全 Mod の処理を打ち切る**:

```
[STOP] レート制限を検出しました（<modid>）。翻訳を中断します。
       制限解除後（通常 5 時間〜）に「抽出 / リソースパック生成」を再度押してください。
```

5 時間 / 週次制限はリトライしても数時間〜1 週間復旧しないため、リトライは時間の浪費でしかない。進捗は `output/.resume/` に保存されているので、ユーザーが時間を空けて再実行すれば続きから処理される (無駄打ちなし)。

### 外側の自動再ループ
全 Mod を走らせた後で `output/.resume/` に未完了が残っていれば、90 秒待機して翻訳全体を再ループ (最大 4 周)。レート制限が原因なら上記の即時中断が効いてこのループには入らない。

### ユーザーによる停止
**「停止」** ボタンを押すと現在のバッチ完了後に止まり、その時点までの進捗はディスクに保存されます。

## 出力されるもの

```text
<出力フォルダ>/<親フォルダ名>_localize/
├── pack.mcmeta      # description: "Generated by MCModLocalizer"
├── pack.png         # アプリ内蔵のアイコンをコピー (既存があれば使用)

```

Minecraft の **「リソースパック」** 画面で有効化することで日本語化が適用されます。

## CLI ツール (`tools/`)

GUI アプリは Mod JAR 内の `en_us.json` を翻訳しますが、それ以外の翻訳ニーズもあります。`tools/` 配下に補助ツールを置いています。

詳細は [tools/README.md](./tools/README.md) を参照。

### `tools/translate_vault_quests.py` — Vault-style 設定ファイル翻訳

Vault Hunters 等の一部 Mod は **クエストブックや UI 文字列を mod 内 lang ではなく、`config/<mod>/lang/<locale>/*.json` で別管理** しています。これを翻訳する CLI:

```powershell
# Vault Hunters のクエストブック等を全部翻訳 (英語ソース → ja_jp)
python tools\translate_vault_quests.py "<instance dir>" --vault-all

# 任意の Vault-style mod を翻訳
python tools\translate_vault_quests.py "<instance dir>" --mod-config config/<mod>

# モッドパック内で Vault-style な mod を自動検出
python tools\translate_vault_quests.py "<instance dir>" --auto-discover
```

特徴:
- GUI アプリと同じ `Provider` 抽象を流用 (Claude SDK / API / Gemini 切替可)
- ファイル単位のチェックポイント (`<出力>.partial`) でレジューム可能
- レート制限ヒット時は即時中断、進捗保持

### `tools/patch_vault_locale.py` — JAR バイナリパッチ

Mod の JAR に `SUPPORTED_LOCALES` がハードコードされていて `ja_jp` が認識されない場合に、**未使用ロケール文字列を `ja_jp` に置換する 5 バイトバイナリパッチ** を当てます:

```powershell
python tools\patch_vault_locale.py "<instance dir>"
```

Vault Hunters Third Edition では `es_mx` (lang ディレクトリが存在しない phantom locale) を `ja_jp` に置換します。コード/ロジック非干渉、`.bak` 自動バックアップ、冪等。

## Vault Hunters の完全日本語化フロー

Vault Hunters Third Edition を例に、上記すべてを組み合わせた手順:

```powershell
# 0. 共通セットアップ (省略: pip install -r requirements.txt + claude login)
$INST = "C:\Users\yukio\curseforge\minecraft\Instances\Vault Hunters Third Edition"

# 1. GUI アプリで Mod JAR の en_us.json を翻訳 → resourcepacks/<...>_localize/
python -m app.main
# Provider=Claude(定額/Code SDK)、Mods/出力フォルダを選んで「抽出/リソースパック生成」

# 2. クエストブック等の config 配下を翻訳 → config/the_vault/lang/ja_jp/
python tools\translate_vault_quests.py $INST --vault-all

# 3. JAR が ja_jp を認識するよう SUPPORTED_LOCALES にパッチ (Minecraft を完全終了してから)
python tools\patch_vault_locale.py $INST

# 4. Minecraft 起動 → 言語=日本語 → ワールドに入る → 完全に日本語化される
```

注意点:
- 手順 3 は **Minecraft / CurseForge を完全に終了** してから(JAR がロックされているとパッチ失敗)
- VH のアップデートで JAR が新しくなったら手順 3 を再実行
- 手順 1 と 2 はレート制限が共通の Claude 定額枠を消費するので時間を空けて段階的に

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| `ModuleNotFoundError: No module named 'flet'` | `pip install -r requirements.txt` を実行 |
| `'Page' object has no attribute 'client_storage'` | Flet を 0.84+ ではなく `flet>=0.28.3,<0.80` に固定する |
| Claude (定額) で `claude-agent-sdk が見つかりません` | `pip install claude-agent-sdk` |
| Claude (定額) で認証エラー | `claude login` で再ログイン |
| Claude (定額) で `[STOP] レート制限を検出しました` | 通常 5 時間後 (週次制限なら週明け) に再実行。詳しくは「中断・再開・自動再ループの挙動」を参照 |
| Gemini が「翻訳が進まない」 | レート制限の可能性。時間を空けて再実行 |
| 翻訳が途中で止まる | `output/.resume/` が残っていれば再実行で続きから処理されます |
| 何が原因で止まったか分からない | `<出力フォルダ>/mcmodlocalizer.log` を確認。GUI ログより詳細な情報が残っています |
| `tools/translate_vault_quests.py` でクエスト書き出しは成功したのに Minecraft で英語のまま | Mod の JAR が `ja_jp` を認識していない可能性。`tools/patch_vault_locale.py` で JAR にロケールを注入する。VH では実証済み |
| `patch_vault_locale.py` が `PermissionError: アクセスが拒否されました` | Minecraft / CurseForge ランチャーが JAR を握っている。完全に終了 (タスクトレイから quit) してから再実行 |
| パッチを当てた Mod が起動しなくなった | `mods/<mod>.jar.bak` を `<mod>.jar` にリネームして元に戻す |
| `tools/translate_vault_quests.py` で「英語ソースが見つかりません」 | Vault-style 構造でない Mod は対応外。`--src-dir` / `--out-dir` で個別パスを指定するか、対象 Mod を諦める |

## アーキテクチャ概要

```
MCModLocalizer/
├── app/                              # GUI アプリ本体 (Flet)
│   ├── main.py                       # エントリポイント
│   ├── ui/app.py                     # Flet GUI (タブ・進捗・自動再開ループ)
│   ├── services/
│   │   ├── extraction.py             # JAR から en_us.json 抽出
│   │   └── translation.py            # 翻訳ワークフロー (resume 管理)
│   └── core/
│       ├── app_logging.py            # ファイルログ (RotatingFileHandler, stdout tee)
│       ├── translation_batch.py      # バッチ実行 (Provider に依存しないリトライ)
│       ├── token_protection.py       # %s, §a, {name} 等の保護/復元
│       ├── chunking.py               # バッチ分割
│       └── llm_providers/
│           ├── base.py               # Provider プロトコル + get_provider()
│           ├── gemini.py             # Gemini OpenAI 互換エンドポイント
│           ├── claude.py             # Anthropic Messages API
│           └── claude_sdk.py         # Claude Agent SDK 経由 (定額プラン)
│                                     # RateLimitExceeded で即時中断
└── tools/                            # 補助 CLI ツール (詳細は tools/README.md)
    ├── translate_vault_quests.py     # Vault-style mod の lang JSON 翻訳
    └── patch_vault_locale.py         # JAR の SUPPORTED_LOCALES に ja_jp 注入
```

### 拡張ポイント

- **新しい Provider を追加**: `Provider` プロトコルを満たすクラスを `llm_providers/` に追加し `base.get_provider()` に登録
- **新しい Vault-style mod の翻訳**: `tools/translate_vault_quests.py --mod-config <path>` でそのまま動作
- **別の mod の JAR ロケールパッチ**: `tools/patch_vault_locale.py` の `OLD` / `NEW` / `CLASS_PATH` 定数を書き換え

### 関連ドキュメント

- [tools/README.md](./tools/README.md) — CLI ツール詳細仕様
- 翻訳機構の流れ: `extraction.py` → `translation.py` → `translation_batch.py` → `llm_providers/<provider>.py`
- レート制限ハンドリング: `claude_sdk.py` の `RateLimitExceeded` → `_translate_targets` → `_run_translation_with_auto_resume`
